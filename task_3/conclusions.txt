Загальні Висновки:

Після вимірювання часу виконання алгоритмів Кнута-Морріса-Пратта (KMP),
Боєра-Мура (Boyer-Moore) та Рабіна-Карпа (Rabin-Karp), можна зробити наступні висновки:

1.Алгоритм Боєра-Мура (Boyer-Moore) зазвичай є найшвидшим серед
 трьох алгоритмів, особливо для великих текстів та пошуку існуючих підрядків.

2.Алгоритм Кнута-Морріса-Пратта (KMP) показує стабільні результати як для
 існуючих, так і неіснуючих підрядків, проте зазвичай повільніший за Боєра-Мура.

3.Алгоритм Рабіна-Карпа (Rabin-Karp) демонструє найповільніші результати, 
особливо для неіснуючих підрядків, що пов'язано з високою вартістю обчислення хешів.

Висновки по текстах:

Текст 1

- Для існуючих підрядків:
    - Алгоритм Боєра-Мура працює швидше за KMP та Рабіна-Карпа.
    - KMP дає середній час виконання, перевершуючи Рабіна-Карпа.
- Для неіснуючих підрядків:
    - Алгоритм Боєра-Мура знову показує найкращі результати.
    - Алгоритм Рабіна-Карпа є найповільнішим.

Текст 2

- Для існуючих підрядків:
    - Алгоритм Боєра-Мура працює швидше, а KMP і Рабіна-Карпа демонструють схожі результати, але KMP все ж працює трохи швидше.
- Для неіснуючих підрядків:
    - Боєр-Мур знову виграє по швидкості.
    - Алгоритм Рабіна-Карпа показує найповільніші результати.

Рекомендації

- Для великих текстів та частого пошуку підрядків алгоритм Боєра-Мура 
    є найкращим вибором завдяки його здатності пропускати великі частини тексту.
- Якщо потрібна стабільність і час пошуку є критичним незалежно від існування
    підрядка, варто розглянути використання **KMP**.
- Алгоритм Рабіна-Карпа може бути корисним для спеціальних випадків,
    наприклад, при пошуку всіх підрядків через хешування, але для класичного пошуку підрядків він програє іншим алгоритмам.
